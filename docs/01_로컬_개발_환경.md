# 01. 로컬 개발 환경 설정

## 1. Node.js 설치 (한 번만)

1. https://nodejs.org 접속
2. **LTS 버전** 다운로드 (큰 초록 버튼)
3. 다운로드된 설치 파일 실행 → "Next" 계속 눌러서 설치
4. 설치 확인: PowerShell 또는 CMD 열고 다음 명령어 입력
   ```
   node --version
   npm --version
   ```
   둘 다 버전 번호가 나오면 성공 (예: `v22.x.x`, `10.x.x`).

## 2. 프로젝트 의존성 설치

PowerShell에서 프로젝트 폴더로 이동해서 실행:

```powershell
cd "C:\Users\MIP James\Downloads\Claude Project\myriad-team-hub"
npm install
```

⏳ 처음 한 번만, 2~3분 걸립니다. (그 뒤로는 다른 사람 컴퓨터에서 실행할 때만 필요)

## 3. 환경 변수 파일 만들기

프로젝트 폴더에 `.env.local` 파일을 만들고 아래 내용을 입력합니다.
(값은 Supabase 가이드 02번에서 얻습니다 — 지금은 비워두거나 건너뛰어도 됨)

```
VITE_SUPABASE_URL=https://xxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOi...
VITE_ALLOWED_EMAIL_DOMAIN=myriadip.com
```

> 💡 `.env.example` 파일을 복사해서 `.env.local`로 이름 바꿔도 됩니다.

## 4. 개발 서버 실행

```powershell
npm run dev
```

성공하면 이런 메시지가 나옵니다:
```
  VITE v7.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

브라우저가 자동으로 http://localhost:5173 을 열어야 합니다.
(Supabase 키를 안 넣었으면 로그인 버튼 눌러도 안 되지만, 화면 구조는 확인 가능)

## 5. 개발 서버 종료

PowerShell 창에서 **Ctrl+C** → `Y` 엔터.

## 자주 묻는 문제

**`npm install` 에러가 나요**
→ 관리자 권한으로 PowerShell 실행해보기. 그래도 안 되면 에러 메시지를 Claude에게 보내주세요.

**`http://localhost:5173` 들어가도 빈 화면이에요**
→ `F12` 눌러서 Console 탭 에러 확인. 대부분 `.env.local` 문제.

**PowerShell에서 `cd` 명령이 안 먹혀요**
→ 경로를 전체 따옴표로 감싸세요: `cd "C:\Users\MIP James\..."`
