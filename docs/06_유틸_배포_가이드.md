# 06. 유틸 배포 가이드 (관리자 전용)

팀이 쓰는 각 유틸 (Enforcement Tools, Market Image Matcher 등)을 웹에서 바로 실행 가능하게 배포하는 절차.

## 개념

```
[관리자 PC]
  유틸 폴더  (EXE + _internal + chromedriver 등)
  ↓ ZIP 압축
  <slug>_v1.0.0.zip
  ↓ 업로드
GitHub Releases
  ↓ 웹 관리자 페이지에 URL 등록
  ↓
[팀원 PC의 런처가]
  실행 요청 시 ZIP 다운로드 → %LOCALAPPDATA%\MyriadLauncher\tools\ 에 해제 → 실행
```

## 유틸 하나당 절차

예시로 **Report Generator** 를 배포한다고 가정:

### 1. 유틸 폴더 ZIP 압축

1. 파일 탐색기에서 `C:\Users\MIP James\Downloads\Report_Generator\` 우클릭
2. **압축(ZIP) 파일로 보내기**
3. 생성된 `Report_Generator.zip` 이름을 **`report-generator_v1.0.0.zip`** 으로 변경
   - 파일명 규칙: `<slug>_v<version>.zip` (팀이 알아보기 쉽게)

> ⚠️ **ZIP 구조 확인**: 압축 파일을 열었을 때 **맨 위에 `Report_Generator\` 폴더가 있고** 그 안에 `Report_Generator.exe` 가 있어야 합니다. 만약 ZIP 열자마자 바로 EXE 가 있는 구조면 `entry_exe` 값이 달라집니다.

### 2. GitHub Releases 에 업로드

1. https://github.com/MIP-James/myriad-oe-team-hub 접속
2. 우측 **Releases** 클릭 → **Draft a new release**
3. **Choose a tag** → `report-generator-v1.0.0` 입력 → "Create new tag" 선택
4. **Release title**: `Report Generator v1.0.0`
5. **Description**: 변경사항 간단히 (선택)
6. 아래 **Attach binaries** 영역에 `report-generator_v1.0.0.zip` 드래그
7. 업로드 완료 기다리기 (파일 크기에 따라 수 분)
8. **Publish release** 클릭
9. 업로드된 ZIP 옆 파일명 **우클릭 → 링크 복사** 또는 **Assets** 섹션에서 URL 확인
   - URL 형식: `https://github.com/MIP-James/myriad-oe-team-hub/releases/download/report-generator-v1.0.0/report-generator_v1.0.0.zip`

> 💡 저장소가 Private 이면 다운로드에 인증이 필요합니다. 팀원 모두 저장소 접근 권한 부여 or 저장소를 Public 으로 전환 중 택일.

### 3. 웹 관리자 페이지에 등록

1. 웹 `/admin/utilities` (관리자만)
2. Report Generator 항목의 **편집**
3. 필드 채우기:
   - **다운로드 URL**: 위에서 복사한 GitHub Releases URL
   - **ZIP 내부 실행 파일 경로 (entry_exe)**: `Report_Generator/Report_Generator.exe`
     - ZIP을 열었을 때 보이는 폴더 구조 그대로 `폴더명/EXE파일명`
   - **버전**: `1.0.0`
   - **이번 버전 변경사항**: "초기 배포" 등
4. **저장**

### 4. 팀원 쪽 검증

본인 PC 의 런처로 테스트:
1. 웹 `/utilities` → Report Generator → **실행**
2. 런처가 자동으로 다운로드 + 해제 (1~2분)
3. 웹 **작업 이력** 페이지에서 진행 상황 실시간 확인:
   ```
   [HH:MM:SS] 설치 상태 확인 중...
   [HH:MM:SS] 'Report Generator' 다운로드 중...
   [HH:MM:SS] 'Report Generator' 다운로드 10% (37MB / 374MB)
   [HH:MM:SS] 'Report Generator' 다운로드 20% ...
   ...
   [HH:MM:SS] 'Report Generator' 압축 해제 중...
   [HH:MM:SS] 'Report Generator' 설치 완료 (v1.0.0)
   ```
4. 본인 PC 에서 Report Generator 실행됨 → 정상 동작 확인
5. 트레이 → **설치된 유틸 폴더 열기** → `tools\report-generator\Report_Generator\` 확인

### 5. 팀원에게 공지

팀 커뮤니티(또는 일단 사내 메신저)로 공지:

> "Report Generator v1.0.0 이 등록됐습니다. 웹 유틸리티 메뉴 → 실행 버튼 누르면 본인 PC에 자동 설치됩니다. 최초 실행은 다운로드 때문에 1~2분 걸려요."

## 새 버전 업데이트

유틸을 새 버전으로 갱신할 때:

1. 로컬에서 업데이트된 유틸 폴더 ZIP 압축 → `report-generator_v1.1.0.zip`
2. GitHub Releases 에 **새 릴리즈** 생성 (태그: `report-generator-v1.1.0`, 새 ZIP 업로드)
3. 웹 `/admin/utilities` → Report Generator 편집:
   - **다운로드 URL**: 새 릴리즈의 ZIP URL
   - **버전**: `1.1.0`
   - **이번 버전 변경사항**: 실제 변경 내역
4. 저장
5. 팀원 쪽은 다음 "실행" 시 런처가 **버전 불일치 감지 → 자동 재다운로드**
   - 기존 설치 폴더 삭제 → 새 버전 다운로드 → 해제 → 실행

## 체크리스트 (유틸 하나 배포 시)

- [ ] 팀원에게 공지

## 파일 크기 주의

- GitHub Releases 는 **파일당 2GB 제한**
- 본 프로젝트 유틸 중 가장 큰 Report Generator 가 374MB → 충분
- 팀 전체 월 다운로드량도 GitHub 제약 없음 (일반 용도 내)

## 문제 해결

**팀원 런처 로그에 "압축 해제 실패"**
- ZIP 파일이 손상됐거나 PowerShell 로 만든 이상한 ZIP 일 수 있음
- 파일 탐색기의 기본 "압축(ZIP) 파일로 보내기" 로 다시 압축
- 또는 7-Zip 같은 도구로 정상 ZIP 생성

**"entry_exe 'xxx' 가 안에 없습니다"**
- 관리자 페이지의 `entry_exe` 경로가 ZIP 구조와 맞지 않음
- ZIP 을 열어서 폴더 구조 확인 후 정확히 입력

**다운로드 URL 이 403 Forbidden**
- 저장소가 Private 인데 팀원이 GitHub 접근 권한 없음
- 해결: 팀원 GitHub 계정을 저장소 collaborator 로 추가 or 저장소 Public 전환
- 또는 Cloudflare R2, Google Drive 등 별도 공개 호스팅 사용
