# 05. 로컬 런처 설치

MYRIAD Launcher를 본인 PC에 설치하면 웹 대시보드의 "실행" 버튼으로 유틸을 원격 실행할 수 있습니다.

## 전체 흐름

```
1. Python 설치 (이미 있으면 건너뛰기)
2. 런처 폴더를 본인 PC에 내려받기
3. setup.bat 더블클릭 → 의존성 설치 + 대화형 설정
4. run.bat 더블클릭 → 런처 시작
5. 웹의 "내 런처" 에서 온라인 상태 확인
```

## 1. Python 설치 확인

PowerShell 열고:

```powershell
python --version
```

- `Python 3.10` 이상이면 OK
- 없으면 https://www.python.org/downloads/ → Download Python → **설치 시 "Add python.exe to PATH" 반드시 체크**

## 2. 런처 파일 준비

팀 GitHub 저장소에서 내려받는 방법:

```powershell
cd "C:\Users\%USERNAME%\Documents"
git clone https://github.com/MIP-James/myriad-oe-team-hub.git
```

`myriad-oe-team-hub\utilities\launcher\` 폴더가 런처 홈입니다.

> 💡 Git 없이 하려면 GitHub 저장소 → Code → Download ZIP → 압축 풀기. 하지만 업데이트는 git clone 이 편합니다.

## 3. 웹에서 연결 토큰 발급

1. 웹 대시보드 로그인: https://myriad-oe-team-hub.pages.dev
2. 왼쪽 **내 런처** 메뉴
3. **토큰 발급** 버튼 → 긴 문자열 생성됨
4. **복사** 버튼 → 클립보드에 저장 (ctrl+c 로 일반 복사도 OK)

> 🔐 **이 토큰은 본인 계정 권한을 런처에 부여합니다. 타인 공유 금지.**
> 실수로 유출된 경우 웹의 "내 런처" → 해당 디바이스 **삭제** → 재발급.

## 4. 설정

`utilities\launcher\setup.bat` 를 **더블클릭**.

진행 단계:
```
[1/2] Python 의존성 설치 중... (2~3분, 처음 한 번만)
[2/2] 대화형 설정 시작...

[1/3] 연결 토큰 붙여넣으세요: <붙여넣기> 엔터
     ✓ 파싱 성공: james@myriadip.com
     ✓ 서버 검증 완료

[2/3] 이 PC를 웹에 어떻게 표시할지?
     디바이스 이름 [james PC]: James 업무PC ← 원하는 이름 입력

[3/3] 각 유틸 EXE 경로:
     🛡️ MYRIAD Enforcement Tools
        경로: C:\Users\MIP James\Downloads\MYRIAD_Enforcement_Tools\MYRIAD_Enforcement_Tools.exe
     🖼️ Market Image Matcher
        경로: (엔터로 스킵 가능)
     ...

✓ config.json 저장됨
```

- **경로 입력 팁:** 파일 탐색기에서 EXE 파일을 Shift+우클릭 → "경로로 복사"
- 지금 경로 모르면 **엔터로 스킵** → 나중에 `utilities\launcher\config.json` 직접 편집

## 5. 런처 실행

`utilities\launcher\run.bat` 더블클릭.

정상 기동 시 콘솔 창에:
```
============================================================
  MYRIAD Launcher v0.1.0
  Device: James 업무PC
============================================================
[14:32:10] Supabase 연결 중...
[14:32:11] 연결 성공: james@myriadip.com
[14:32:11] 디바이스 등록 완료: James 업무PC (id=abc12345...)
[14:32:11] 폴링 시작 (interval=3.0s). Ctrl+C 로 종료.
```

웹 대시보드 **내 런처** → 연결된 PC 목록에 **초록 점 + 온라인** 뱃지 뜨면 성공 ✅

## 6. 실제 동작 테스트

1. 웹 → **유틸리티** 메뉴
2. 우측 상단 "**1개 런처 온라인**" 초록 뱃지 확인
3. 아무 유틸 카드의 **실행** 버튼 클릭
4. 약 3~5초 뒤 본인 PC에서 해당 EXE 가 실행됨
5. 웹의 **작업 이력** 메뉴에서 상태가 대기 → 실행 중 → 완료 로 흘러가는 것 확인

## 종료

런처 콘솔 창에서 **Ctrl+C** → 웹에서 오프라인 뱃지로 전환.

창을 그냥 닫으면 오프라인 업데이트가 안 될 수 있지만, 30초 뒤 하트비트 누락으로 자동 감지됩니다.

## 자주 묻는 문제

**`'python' is not recognized`**
→ Python이 PATH에 없음. Python 재설치 시 "Add to PATH" 체크.

**`pip install` 실패 (SSL error, proxy 등)**
→ 회사망 프록시 문제. IT팀에 pypi 접근 허용 요청, 또는 `pip --proxy=http://...` 지정.

**토큰 검증에서 "토큰이 유효하지 않습니다"**
→ 웹에서 토큰을 재발급한 뒤 다시 시도. (토큰은 발급 후 곧바로 써야 안전)

**실행 버튼을 눌렀는데 아무 반응이 없음**
→ 런처 콘솔 창에 로그 확인:
- `[오류] 경로가 설정되지 않았습니다` → `config.json` 의 `utility_paths` 에 해당 slug 추가
- `[오류] EXE 파일을 찾을 수 없습니다` → 경로 오타 또는 파일 이동됨

**PyInstaller로 단일 EXE 로 만들기**
→ 추후 제공. 현재는 Python 스크립트 직접 실행.

## 팀 전체 배포 시

각 팀원이 이 가이드를 따라 개별 설치합니다.
- 토큰은 **개인별**로 발급되어야 함
- 한 토큰을 여러 PC에서 쓰면 중복 실행/상태 충돌 가능

관리자가 각 팀원에게 이 가이드를 공유하고, 초기 설치에 1:1 지원하면 수월합니다.
