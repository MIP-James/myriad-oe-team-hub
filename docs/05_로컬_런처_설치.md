# 05. 로컬 런처 설치 (팀원용)

MYRIAD Launcher 를 본인 PC에 설치하면 웹 대시보드의 "실행" 버튼으로 유틸을 바로 실행할 수 있습니다.

**좋은 소식**: 유틸은 **자동 다운로드** 되므로, 런처 하나만 설치하면 끝입니다. 각 유틸 EXE 를 따로 구하거나 경로 설정할 필요 없습니다.

## 배포 파일

관리자가 주는 파일:

```
MyriadLauncher\
├── MyriadLauncher.exe   ← 트레이 상주 런처
└── MyriadSetup.exe       ← 최초 설정
```

두 파일은 **반드시 같은 폴더에** 두세요.

## 1. 토큰 발급

1. 브라우저로 https://myriad-oe-team-hub.pages.dev 로그인
2. 왼쪽 **내 런처** 메뉴
3. **토큰 발급** 클릭 → **복사** 버튼 (클립보드 저장됨)

> 🔐 이 토큰은 **본인 계정 권한**을 런처에 부여합니다. 타인 공유 금지.

## 2. 설정 (MyriadSetup.exe)

`MyriadSetup.exe` **더블클릭**. 첫 실행은 압축 해제 때문에 5~10초 걸립니다.

콘솔에서 물어보는 항목:
```
[1/2] 토큰 붙여넣기: <Ctrl+V 또는 우클릭 붙여넣기>
     ✓ 검증 완료
[2/2] 디바이스 이름:  James 업무PC

✓ 설정 저장됨
유틸리티 자동 다운로드: 실행 요청 시 런처가 최초 1회 자동 설치합니다.
```

## 3. 런처 실행

`MyriadLauncher.exe` **더블클릭**.

- 콘솔 창 **안 뜸**. 화면 우측 하단 **시스템 트레이**의 `∧` 을 펼치면 주황 M 아이콘 있음
- 우클릭 메뉴에서 "**Windows 시작 시 자동 실행**" 체크 권장 (부팅 시 자동 시작)

## 4. 실제 실행

1. 웹 → **유틸리티** 메뉴
2. 우측 상단 "**1개 런처 온라인**" 초록 뱃지 확인
3. 아무 유틸 카드의 **실행** 버튼 클릭
4. **최초 실행 시**: 런처가 자동으로 해당 유틸 다운로드 + 설치 (1~2분, 유틸 크기에 따라)
   - `%LOCALAPPDATA%\MyriadLauncher\tools\<유틸이름>\` 에 설치됨
   - 진행률이 웹의 **작업 이력** 페이지에 실시간 표시됨
5. 설치 완료되면 바로 실행
6. **두 번째 실행부터는** 다운로드 스킵, 바로 실행

## 트레이 메뉴

| 메뉴 | 설명 |
|---|---|
| 🟢 상태 | 연결/대기/실행 중 (정보) |
| 💻 디바이스 이름 | 현재 등록된 이름 |
| 📌 최근 작업 | 직전 실행 결과 |
| 웹 대시보드 열기 | 기본 브라우저로 이동 |
| 설치된 유틸 폴더 열기 | `%LOCALAPPDATA%\MyriadLauncher\tools\` |
| 로그 보기 | `launcher.log` 열기 |
| 토큰 / 설정 재설정 | MyriadSetup 재실행 |
| Windows 시작 시 자동 실행 | 시작프로그램 등록 토글 |
| 종료 | 런처 정상 종료 |

## 문제 해결

**트레이 아이콘 안 보임**
- 작업표시줄 트레이의 `∧` 펼치기
- 없으면 `MyriadLauncher.exe` 더블클릭
- 같은 폴더의 `launcher_error.log` 확인 (있으면 관리자에게 전달)

**상태가 "인증 실패 (토큰 재설정 필요)"**
- 트레이 우클릭 → **토큰 / 설정 재설정**
- 웹 `/launcher` 에서 새 토큰 발급 → 붙여넣기

**"실행" 눌렀는데 "자동 설치 실패" 에러**
- 관리자가 아직 해당 유틸의 **다운로드 URL 을 웹에 등록 안 함**
- 관리자에게 "xx 유틸 다운로드 URL 등록해주세요" 요청
- 관리자가 등록하면 런처 재시작 없이 바로 "실행" 다시 누르면 됨

**설치는 됐는데 EXE 파일 못 찾음**
- 트레이 → **설치된 유틸 폴더 열기** → 해당 유틸 폴더 안에 EXE 있는지 확인
- 경로 문제면 관리자에게 `entry_exe` 설정 확인 요청

**수동으로 특정 경로 지정하고 싶음 (power user)**
- 런처를 종료
- `MyriadSetup.exe` 가 있는 폴더의 `config.json` 열기
- `"utility_paths"` 에 다음 형식으로 추가:
  ```json
  "utility_paths": {
    "report-generator": "C:\\내경로\\Report_Generator.exe"
  }
  ```
- 수동 경로가 있으면 자동 다운로드 대신 해당 경로 사용
