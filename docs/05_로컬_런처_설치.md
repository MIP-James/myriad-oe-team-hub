# 05. 로컬 런처 설치

MYRIAD Launcher를 본인 PC에 설치하면 웹 대시보드의 "실행" 버튼으로 유틸을 원격 실행할 수 있습니다. 트레이 아이콘으로 조용히 동작하며, PC 부팅 시 자동 시작 설정도 가능합니다.

## 배포 형태 (팀원 관점)

관리자가 한 번 빌드한 뒤 팀원에게 두 개 파일만 주면 됩니다:

```
MyriadLauncher\
├── MyriadLauncher.exe   ← 트레이에 상주하며 실행 처리
├── MyriadSetup.exe       ← 최초 설정 / 토큰·경로 재설정
```

팀원은 Python 설치 불필요. 그냥 폴더 복사 + 실행.

---

## [관리자] EXE 빌드 방법

빌드는 **1회만** 하면 되고, 이후 팀원에게 `dist/` 의 두 EXE를 배포합니다.

### 1. Python 3.10+ 설치 (빌드 환경에만 필요)

https://www.python.org/downloads/ → LTS 버전 → "Add python.exe to PATH" 체크

### 2. 빌드

```powershell
cd "C:\Users\MIP James\Downloads\Claude Project\utilities\launcher"
build.bat
```

자동 진행:
1. PyInstaller 및 의존성 설치
2. `launcher.ico` 아이콘 자동 생성 (주황 원 + M 글자)
3. **MyriadLauncher.exe** 빌드 (트레이/콘솔 없음)
4. **MyriadSetup.exe** 빌드 (콘솔 기반 설정)

⏳ 5~10분 소요. 완료되면 `dist\` 폴더에 두 EXE 생성.

### 3. 배포

`dist\MyriadLauncher.exe` + `dist\MyriadSetup.exe` 두 파일을 **같은 폴더에** 넣어서 팀원에게 전달 (사내 드라이브, 이메일 첨부 등).

---

## [팀원] 설치 & 첫 실행

### 1. 파일 배치

관리자에게 받은 두 EXE를 원하는 폴더에 복사. 예:
```
C:\MyriadLauncher\
├── MyriadLauncher.exe
└── MyriadSetup.exe
```

> ⚠️ 두 EXE는 **반드시 같은 폴더에** 있어야 서로 호출 가능.

### 2. 웹에서 연결 토큰 발급

1. 브라우저에서 https://myriad-oe-team-hub.pages.dev 로그인
2. 왼쪽 **내 런처** 메뉴
3. **토큰 발급** → **복사** 버튼 클릭 (클립보드에 저장됨)

> 🔐 토큰은 본인 계정 권한을 런처에 부여하니 **타인 공유 금지**.
> 유출 시 웹 "내 런처" 에서 해당 디바이스 **삭제** → 재발급.

### 3. 설정 (MyriadSetup.exe)

`MyriadSetup.exe` **더블클릭** → 콘솔 창이 뜨면서 순서대로 물어봅니다.

```
[1/3] 토큰 붙여넣으세요: <Ctrl+V 또는 우클릭 붙여넣기>
     ✓ 파싱 성공
     ✓ 서버 검증 완료

[2/3] 디바이스 이름:  James 업무PC ← 웹에 이렇게 표시됨

[3/3] 각 유틸 EXE 경로 입력
     (엔터 = 스킵, 나중에 설정 가능)

  🛡️ MYRIAD Enforcement Tools
     경로: C:\...\MYRIAD_Enforcement_Tools.exe
  🖼️ Market Image Matcher
     경로: C:\...\MarketImageMatcher.exe
  ...

✓ 설정 저장됨
```

> 💡 **경로 입력 팁:** 파일 탐색기에서 EXE 파일 **Shift+우클릭** → "경로로 복사".

### 4. 런처 실행 (MyriadLauncher.exe)

`MyriadLauncher.exe` **더블클릭**.

- 콘솔 창 **안 뜸**
- 화면 우측 하단 **시스템 트레이**에 **주황 원에 M** 아이콘이 생성됩니다
- ⚠️ 트레이 아이콘이 안 보이면: 트레이의 **∧** (숨겨진 아이콘) 버튼 클릭

### 5. 트레이 아이콘 사용

아이콘 **우클릭**:
```
🟢 온라인 - 대기 중
💻 James 업무PC
📌 최근: (아직 없음)
─────────────────────────
웹 대시보드 열기
로그 보기
토큰 / 경로 재설정
✓ Windows 시작 시 자동 실행   ← 체크박스
─────────────────────────
종료
```

**처음엔 "Windows 시작 시 자동 실행" 클릭** — 부팅 시 자동 실행됩니다 (체크 표시 생김).

### 6. 동작 확인

1. 웹 → **유틸리티** 메뉴
2. 우측 상단 "**1개 런처 온라인**" 초록 뱃지 확인
3. 아무 유틸 카드 **실행** 버튼 클릭
4. 3~5초 뒤 본인 PC에서 해당 EXE 실행
5. 웹의 카드 상태가 `대기 → 실행 중 → 완료` 변화
6. 트레이 아이콘 우클릭 → "최근: OO 완료" 표시

## 트레이 메뉴 상세

| 메뉴 | 설명 |
|---|---|
| **🟢/⚪ 상태** | 현재 연결 상태 (토글 불가, 정보만) |
| **💻 디바이스 이름** | 현재 등록된 이름 |
| **📌 최근 작업** | 직전 실행 결과 요약 |
| **웹 대시보드 열기** | 기본 브라우저로 바로 이동 |
| **로그 보기** | `launcher.log` 를 기본 텍스트 에디터로 열기 |
| **토큰 / 경로 재설정** | MyriadSetup.exe 재실행 |
| **Windows 시작 시 자동 실행** | 시작프로그램 등록/해제 토글 |
| **종료** | 런처 정상 종료 + 웹에 오프라인 표시 |

## 문제 해결

### 트레이 아이콘이 안 보임
- 작업표시줄 트레이 영역 **∧** 클릭 → 숨겨진 아이콘 확인
- 없으면 `MyriadLauncher.exe` 다시 실행
- 같은 폴더의 `launcher_error.log` 확인 (있으면 에러 내용 복사)

### 트레이 상태가 "인증 실패 (토큰 재설정 필요)"
- 메뉴 → **토큰 / 경로 재설정** → 웹에서 새 토큰 발급 → 붙여넣기

### 실행 버튼 눌러도 반응 없음
- 트레이 메뉴 → **로그 보기** → 최근 항목 확인
- 흔한 원인:
  - `[ERROR] 경로 미설정` → 메뉴 → 토큰/경로 재설정 → 해당 유틸 경로 추가
  - `[ERROR] EXE 파일 없음` → 경로 오타 또는 파일 이동

### 런처를 업데이트하고 싶음
- 새 EXE 두 개로 덮어쓰기 → 같은 `config.json` 이 재사용됨

### 자동 실행이 안 됨
- 트레이 메뉴에서 한 번 해제 → 다시 체크 (바로가기 재생성)
- `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\` 폴더 확인

### 여러 PC에서 같은 계정 쓰고 싶음
- 각 PC에서 **각각 별도로 토큰 발급 & 설정**. 한 토큰을 여러 PC에서 쓰면 상태 충돌.
- 웹 "내 런처" 에 디바이스가 여러 개 뜸 (디바이스 이름으로 구분)

## 개발 모드로 실행 (Python 직접 실행, 관리자/디버깅용)

빌드 없이 로컬에서 바로 테스트하려면:

```powershell
cd "C:\Users\MIP James\Downloads\Claude Project\utilities\launcher"
setup.bat     # 처음 한 번
python myriad_launcher.py
```

`pythonw.exe` 로 실행하면 콘솔 없이 트레이만 뜹니다:
```powershell
pythonw myriad_launcher.py
```
